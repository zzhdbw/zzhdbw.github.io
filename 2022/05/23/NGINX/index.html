一, 基本概念

nginx是什么?

Nginx 是高性能的 HTTP 和反向代理的服务器，处理高并发能力是十分强大的，能经受高负载的考验,有报告表明能支持高达 50,000 个并发连接数。

正向代理

1,需要在客户端配置代理服务器进行指定网站访问

反向代理

1,暴露的是代理服务器地址，隐藏了真实服务器IP地址。

负载均衡

将大量的请求平均分发到多个不同的服务上

动静分离

二, 软件安装和配置

linux nginx安装

1,安装pcre依赖(以下软件均在usr/src中安装)

  • 联网下载 pcre 压缩文件依赖
  • wget http://downloads.sourceforge.net/project/pcre/pcre/8.37/pcre-8.37.tar.gz
  • 解压压缩文件
  • 使用命令 tar -xvf pcre-8.37.tar.gz
  • 进入解压的文件夹后 执行./configure命令 如果提示缺依赖就先执行2

  • yum -y install make zlib zlib-devel gcc-c++ libtool openssl openssl-devel

3,安装nginx

  • 使用命令解压tar –xvf nginx***

  • ./configure

  • make && make install

  • 进入目录 /usr/local/nginx/sbin/ 启动服务

四, ngnix配置实例-负载均衡

实现访问10.3.224.3:9090/edu/a.html 平均分配到8080 和 8081端口中

负载均衡的分配策略:

  • 1,轮询 按照请求的时间顺序分配 失效的机器可以自动剔除

  • 2,weight 根据权重分配 权重越大被分配的机会越大

  • 3,ip_hash 每个访问按照请求者的ip进行分配,每个访客无论何时请求的都是同一个后端服务器，可以解决session的问题

  • 4,fair(第三方) 按照后端服务器的响应时间分配请求,相应时间短的优先分配

五, ngnix配置实例-动静分离

让访问静态资源的请求转发到静态资源服务器,请求动态资源的转发到tomcat服务器，这样可以减少tomcat的压力，提高tomcat性能。

步骤:

首先创建文件夹data/www 和data/image

接着配置nginx配置:

server {
    listen       80;
    server_name  localhost;
    #charset koi8-r;

    #access_log  logs/host.access.log  main;

    location /www/ {
        root   /data/;
        index  index.html index.htm;
    }
    location /image/ {
        root   /data/;
        autoindex on;
    }

即可实现效果:

http://10.3.224.3/image/01.jpg 访问的是/data/image/01.jpg

实现了静态资源的部署。

七, nginx 原理

工作原理

一个 master 和多个 woker 的好处

  • 可以使用nginx –s reload 热部署，利用 nginx 进行热部署操作

  • 每个 woker 是独立的进程，如果有其中的一个 woker 出现问题，其他 woker 独立的，继续进行争抢，实现请求过程，不会造成服务中断

设置多少个 woker 合适

  • worker 数和服务器的 cpu 数相等是最为适宜的

连接数worker_connection

  • 发送请求会占用worker的几个连接数？

    答案:2个或4个 访问静态资源就是两个 再访问tomcat就是4个

假设有5个worker，一个master,每个worker支持最大连接数1024,支持的最大并发量是多少?

  • 普通静态访 问最大并发数是:5*1024/2 = 2560
  • 若是http做反向代理,最大并发数是5*1024/4 = 1280

开启防火墙service firewalld start

关闭防火墙service firewalld stop

开放防火墙80端口firewall-cmd –zone=public –add-port=80/tcp –permanent

  • –zone=public 指定作用域为public
  • –add-port=80/tcp放开80端口 协议为tcp
  • –permanent永久生效

重新加载防火墙使配置生效firewall-cmd –reload

查看已经开放的端口 firewall-cmd –list-ports

关闭已经开放的端口 firewall-cmd –permanent –remove-port=80/tcp

批量开放端口 开放80-90之间的所有端口firewall-cmd –zone=public –add-port=80-90/tcp –permanent

批量关闭端口firewall-cmd –remove-port=80-90/tcp –permanent
